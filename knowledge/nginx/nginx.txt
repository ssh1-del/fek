什么是同源？
同源即两个界面的接口协议，端口号，域名都相同

什么是同源策略?
同源策略就是浏览器的安全限制，阻止了不同域之间的数据交互
目的：所谓的同源策略都是由浏览器所限制的，在于保护用户的隐私安全问题

安全限制具体都组织了那些东西不可以被访问？
1.无法读取非同源下的cookie，localStorage，indexDB
2.无法解除非同源的dom
3.无法向非同源发送axios请求（准确的说是可以发送请求，但是会被浏览器拦截）

如果浏览器没有同源策略的限制，出现的安全隐患有哪些？
1.防止恶意网站获取其他网站的cookie数据。
   （1）浏览器cookie一般存储什么信息？
    cookie存储的内容是由服务端生成的，目的在于方便用户下次登陆，
    可以通过cookie来回去session信息，cookie保存的是服务端返回的字符串，session保存的是对象
2.如果可以获取dom，恶意网站可以通过iframe打开银行的网页，可以获取整个银行页面的信息
3.发送非同源的ajax的请求，比如你在网站上登陆过淘宝，就可以通过cooki直接登陆，获取信息

什么是反向代理？（客户端不明确目标服务器）
就像你找代购买东西，你把你想要买的东西告诉代购，然后代购帮你去买，最终的结果是代购帮你买到了你想要的东西，
但是你不知道这个东西从哪里买的。
nginx就是http服务器，也就是反向代理服务器。反向代理代理服务器位于用户和目标服务器之间，对于用户来讲，
反向代理服务器就相当于目标服务器，即用户直接访问反向代理服务器就可以获得目标服务器的资源。用户其实不知道目标服务器的地址

什么事负载均衡？
公司会建立很多的服务器，并且在服务器集群中的每一台服务的内容是一样的，然后当用户去访问服务器的时候，先访问一个代理服务器，
然后在让代理服务器选择一个压力比较小的服务器让用户去访问，从而保证在服务集群里面的每一个服务的压力趋于平衡，避免服务崩困

什么是正向代理？（客户端明确目标服务器）
客户端发送请求到目标服务器的过程中，中间有一个代理服务器，客户端先请求代理服务器，然后代理服务器转发请求到目标服务器
然后目标服务器将结果发送给代理服务器，代理服务器在将结果转发给客户端，在整个流程中，客户端明确目标服务器是谁


Mac环境先安装配置Nginx

使用brew安装nginx  命令：brew install nginx
加入开机启动nginx  命令： sudo cp /usr/local/opt/nginx/homebrew.mxcl.nginx.plist /Library/LaunchDaemons/
                       sudo launchctl load -w /Library/LaunchDaemons/homebrew.mxcl.nginx.plist
启动ngingx  命令：sudo nginx 
停止nginx服务 命令： sudo nginx -s stop
重启nginx服务 命令： suod nginx -s reload   每次修改nginx.config文件后需要重启nginx
检查nginx.config文件是否配置正确 命令： nginx -t  如果出现OK，successful说明配置正确
