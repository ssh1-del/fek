useEffect

使用方式：该钩子包含命令式函数，且可能有副作用代码的函数。

出现useEffect的原因：
   在函数组件主体内（这里指在 React 渲染阶段）改变 DOM、添加订阅、设置定时器、记录日志以及执行其他包含副作用的操作都是不被允许的，
   因为这可能会产生莫名其妙的 bug 并破坏 UI 的一致性。
   使用 useEffect 完成副作用操作。赋值给 useEffect 的函数会在组件渲染到屏幕之后执行。你可以把 effect 看作从 React 的纯函数式世界通往命令式世界的逃生通道。
   useEffect是在每次渲染后执行(和commonDidMount相似)

useEffect的第一个参数的返回值
   副作用可以使组件加载发生变化，那么组件卸载的时候需要清理这些效应。
   useEffect允许第一个函数返回一个函数，在组件卸载时执行该函数，清理副效用

什么是纯函数？
   相同的输入，永远会有相同的输出，而且没有任何可观察的副作用，该函数被称为纯函数。
什么是副作用函数？
   一个函数如果在运行时，除了有返回值，还会对主调用函数产生附加影响，被称为副作用函数。
